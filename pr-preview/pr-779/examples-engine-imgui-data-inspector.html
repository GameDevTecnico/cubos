Data Inspector

Using the cubos::engine::DataInspector resource in combination with ImGui integration plugin to inspect/modify data in real time.

See also

Full source code here.

As you already know, the ImGui plugin allows you to integrate the Dear ImGui library with CUBOS., making it quick, straightforward, and effortless to create user interfaces and interactive debugging tools within your applications (check more info here).

In this tutorial, we'll show you how you can inspect/modify data in real time using ImGui integration plugin + cubos::engine::DataInspector resource.

To start off, we'll need to have some sort of dummy data shared across our application, so we can inspect/modify later. Let's create a DummyResource with some fields and fill it with random data.

Note

In case you did not read about Reflection reflection yet, you should take a look at that first, since we'll use some of it's concepts here.

struct Person
{
    CUBOS_REFLECT;
    // @todo stdstring name
    int32_t age;
    float weight;
    bool dead;
};

CUBOS_REFLECT_IMPL(Person)
{
    return cubos::core::reflection::Type::create("Person").with(cubos::core::reflection::FieldsTrait()
                                                                    .withField("age", &Person::age)
                                                                    .withField("weight", &Person::weight)
                                                                    .withField("dead", &Person::dead));
}

struct DummyResource
{
    int integer;
    Person person;
    std::vector<int32_t> vec;
    std::map<int32_t, int32_t> map;
};
static void fillDumyResourceSystem(Write<DummyResource> data)
{
    data->integer = 1337;
    data->person = Person{1, 2.0, false};
    data->vec = {12, 59, 25};
    data->map = {
        {1, 2},
        {2, 4},
        {3, 6},
        {4, 8},
    };
}
    cubos.addResource<DummyResource>();
    cubos.startupSystem(fillDumyResourceSystem);

Well now, using the cubos::engine::DataInspector is pretty easy, all you have to do is, obviously, access the resource on your system, and use the functions DataInspector::show / DataInspector::edit, depending on your needs.

static void exampleDataInspectorSystem(Write<DataInspector> inspector, Read<DummyResource> data)
{
    static auto& integerType = reflect<int>();
    static auto& personType = reflect<Person>();
    static auto& vecType = reflect<std::vector<int32_t>>();
    static auto& mapType = reflect<std::map<int32_t, int32_t>>();

    ImGui::Begin("Data Inspector");
    ImGui::BeginTable("id1", 2);

    inspector->show("data->integer", integerType, &data->integer);
    inspector->show("data->person", personType, &data->person);
    inspector->show("data->vec", vecType, &data->vec);
    inspector->show("data->map", mapType, &data->map);

    ImGui::EndTable();
    ImGui::End();

    ImGui::Begin("Data Modifier");
    ImGui::BeginTable("id2", 2);

    inspector->edit("data->integer", integerType, (void*)&data->integer);
    inspector->edit("data->person", personType, (void*)&data->person);
    inspector->edit("data->vec", vecType, (void*)&data->vec);
    inspector->edit("data->map", mapType, (void*)&data->map);

    ImGui::EndTable();
    ImGui::End();
}

Note

The DataInspector::show and DataInspector::edit functions require an argument of type reflection::Type in order to accurately represent or modify the value, thats why we first get the object types via reflection.
